UI Enhancement Recommendations for FinPlan 
 
1. Visual Hierarchy & Layout 
Current Issues: 
- Dense information presentation with insufficient visual breathing room 
- Flat visual hierarchy makes it hard to distinguish primary vs. secondary information 
- Navigation sidebar competes with main content for attention 
 
 
2. Navigation & Process Flow 
Current Issues: 
- 12-step process shown as a horizontal list is hard to track 
- "PROCESS" and "CALCULATORS" sections in sidebar are visually similar 
- Current step is not prominently highlighted 
 
Recommendations: 
A. Redesign Progress Indicator: 
- Replace horizontal step list with a vertical stepper on desktop (left sidebar) showing: 
- Completed steps with checkmarks 
- Current step highlighted with accent color 
- Upcoming steps grayed out 
- On mobile: Use a collapsed progress bar with step count (e.g., "Step 3 of 12") 
 
B. Sticky Navigation: 
- Make the "Back to..." and "Proceed to..." buttons sticky at the bottom of the viewport 
- Add a mini-progress indicator above these buttons 
C. Calculator Section: 
- Move calculators to a separate "Tools" section or collapsible drawer 
- Use icons with tooltips instead of a full text list 
 
3. Form Design & Input Fields 
Current Issues: 
- Multiple input fields laid out in a grid without clear groupings 
- "Configure Details" links don't indicate what they configure 
- Currency inputs lack proper formatting feedback

4. Data Tables & Timelines 
Current Issues: 
- Year-wise tables (Journey, Allocation, Net Worth) are very wide and hard to scan 
- No visual cues for important data points 
- Monotonous rows without differentiation 
Recommendations: 
A. Table Enhancements: 
- Add alternating row colors (`bg-muted/50` on even rows) 
- Highlight current year with accent border-left 
- Add sticky first column (Year) on horizontal scroll 
- Use conditional formatting: green for positive values, red for deficits 
- Add sparkline mini-charts in summary rows 
 
B. Timeline Visualization: 
- Add a visual timeline bar above data tables showing major milestones (retirement, 
children's education, etc.) 
- Use milestone markers for goal target years 
 
C. Pagination/Virtualization: 
- For tables with 20+ years, add "Show 5 years | 10 years | All" toggle 
- Consider collapsible decade groups (2026-2030, 2031-2040, etc.) 
 
5. Color Palette & Theming 
Current Issues: 
- Heavy use of default blue which feels generic 
- Insufficient color differentiation between data categories 
- Low contrast on some secondary text 
Recommendations: 
Primary Palette (limit to 5 colors): 
 
 
Apply consistently: 
- Protection Gap: Use amber cards with warning icon 
- Fully Funded Goals: Use emerald checkmark badge 
- Coverage shortfall: Use rose text with exclamation 
 
6. The Financial Pyramid (Protection Gap Module) 
Current Issues: 
- Pyramid visualization is static and doesn't feel interactive 
- Layer status not immediately clear 
Recommendations: 
- Make pyramid layers clickable with smooth animations 
- Add glow effect on active layer 
- Show progress percentage on each layer 
- Use iconography for each layer: 
- Foundation: Shield icon 
- Growth: TrendingUp icon 
- Succession: Users icon 
- Add tooltips explaining each layer on hover 
 
7. Family Profile & Analysis Cards 
Current Issues: 
- Family member cards are text-heavy 
- No visual distinction between Self, Spouse, Child 
 
Recommendations: 
- Add avatar placeholders with initials (colored by role) 
- Use role badges: "Self" (primary), "Spouse" (secondary), "Child" (accent) 
- Show age progress bar (current age → retirement age) 
- Add subtle icons for occupation type (briefcase, graduation-cap, etc.) 
- Use card grid instead of vertical stack on desktop 
 
8. Insurance & Coverage Tables 
Current Issues: 
- Year-wise coverage tables are overwhelming 
- Premium/coverage data lacks visual summary 
- Maturity schedule is buried 
Recommendations: 
A. Summary Dashboard: 
- Add a summary card row at top: 
- Total Annual Premium: `X` 
- Total Coverage: `Y` 
- Years until maturity: `Z` 
- Use donut chart for coverage distribution by family member 
B. Table Improvements: 
- Default to collapsed view (show only next 5 years) 
- Add "Download as PDF" option for full tables 
- Use heat map coloring for coverage intensity 
C. Maturity Highlight: 
- Create a dedicated "Upcoming Maturities" card with countdown badges 
 
9. Goal Fulfillment Roadmap (Module 11) 
Current Issues: 
- Goals with multiple years (Higher Education Year 1-6) create repetition 
- "Pending Needed" vs "Assigned" could be clearer 
- Too many input fields visible at once 
Recommendations: 
- Group multi-year goals into expandable accordions (e.g., "Higher Education - Radhika (6 
years)" with sub-items) 
- Use progress bars for funding status (green = assigned, gray = pending) 
- Add "Quick Allocate" button that auto-distributes from available funds 
- Show fund source cards as selectable chips rather than inline inputs 
- Add "Fully Funded" celebration animation (confetti or checkmark pulse) 
 
10. Mobile Responsiveness 
 
 
11. Micro-interactions & Feedback 
Recommendations: 
- Form Saves: Add auto-save indicator ("Saved 2 seconds ago") 
- Button States: Add loading spinners on calculation buttons 
- Transitions: Use `transition-all duration-200` on hover states 
- Tooltips: Add info icons (`HelpCircle`) with explanatory tooltips on complex terms (HLV, Net 
Investible Surplus) 
- Success States: Show toast notifications on successful submissions 
 
12. Accessibility Improvements 
Recommendations: 
- Ensure 4.5:1 contrast ratio for all text 
- Add aria-labels to all interactive elements 
- Implement keyboard navigation for form sections 
- Add focus-visible rings on all interactive elements 
- Use semantic HTML (`<section>`, `<article>`, `<nav>`) 
- Add skip links for main content 
 
Priority Implementation Order 
 
1. High Impact, Quick Wins: Color palette, spacing, card shadows, button styling 
2. Medium Effort: Navigation redesign, form grouping, table improvements 
3. Larger Effort: Financial pyramid interactivity, goal roadmap redesign, mobile optimization
